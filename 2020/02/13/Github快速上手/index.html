SSH公私钥的基本使用

(已注册Github用户后)

创建密钥

用以下命令生成公私钥

$ ssh-keygen

Enter file in which to save the key字样处输入密钥文件的保存路径，直接回车键为默认路径

Enter passphrase及Enter same passphrase设置密码，一路回车跳过……



~/.ssh/路径下，后缀为.pub为公钥文件，无后缀为私钥文件。

关联SSH公钥到Github账户

点击Github右上角头像

Settings -> SSH and GPG keys -> New SSH key ->Title任意填写，复制公钥文件的内容至Key -> Add SSH key



验证连接

打开Git Bash

$ ssh -T git@github.com

出现”Are you sure…”,输入yes,回车

显示 “Hi xxx! You’ve successfully……” 即连接成功。

配置用户名与邮箱

安装完Git工具以后

$ git config --global user.name "你的用户名"

$ git config --global user.email "你的邮箱"

Github及Git工具的基本使用

克隆仓库到本地PC

找到你感兴趣的项目，获得仓库地址。
在Git Bash中键入以下命令：

$ git clone 仓库地址

添加

$ git add <文件名|路径>

删除

$ git rm <文件名|路径>

撤销

$ git reset

提交仓库的改动

$ git commit -m "你的提交备注"

推送改动到远程仓库

$ git push origin <分支名>

使用-u参数之后会绑定这一次的仓库分支名，下一次推送不需要加上分支名。eg:

$ git push -u origin master

查看仓库改动

git fetch origin

$ git fetch

下拉仓库同步

git pull origin <分支名>

$ git pull
